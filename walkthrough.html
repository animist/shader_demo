各デモの逐行解説

実際のデモのコードを頭から読み解く。3本（インク・万華鏡・反応拡散）はチャンクごとに、残りは「勘所」だけを抜き出す。GLSL入門の関数表を隣に開いておくと読みやすい。

demo 1

インク — 全ピクセルがノイズで色を決める

ink.html。フラグメントシェーダーの王道。「座標→ノイズ→色」を組み立てる。

uniform float uTime;  uniform vec2 uResolution;  uniform vec2 uTilt;
uniform float uWarp, uSpeed, uTiltK, uInk, uGlow;  uniform vec3 uBase, uTint;
入力。 時間・解像度・傾きに加え、⚙パラメータ（うねり量 uWarp、速度、傾き感度、インクの広がり、発光、地色、パレット）。
float hash(vec2 p){ p=fract(p*vec2(123.34,345.45)); p+=dot(p,p+34.345); return fract(p.x*p.y); }
疑似乱数。 座標を入れると 0..1 のバラついた値が返る。fract と dot で散らす。sin を使わないのはモバイルで壊れにくくするため（学習ノートの落とし穴）。
float noise(vec2 p){ vec2 i=floor(p), f=fract(p);
  float a=hash(i), b=hash(i+vec2(1.0,0.0)), c=hash(i+vec2(0.0,1.0)), d=hash(i+vec2(1.0,1.0));
  vec2 u=f*f*(3.0-2.0*f); return mix(mix(a,b,u.x), mix(c,d,u.x), u.y); }
value noise。 格子点（整数座標）の乱数を、間の位置 f でなめらかに補間。f*f*(3.0-2.0*f) は角を取る曲線。mix 3回で2次元補間。
float fbm(vec2 p){ float v=0.0, a=0.5; for(int i=0;i<5;i++){ v+=a*noise(p); p*=2.0; a*=0.5; } return v; }
fractal noise（雲）。 ノイズを「細かく・薄く」しながら5回重ねる（p*=2で倍の細かさ、a*=0.5で半分の濃さ）。これで自然な濃淡になる。
vec3 pal(float t){ return 0.5 + 0.5*cos(6.28318*(t + uTint)); }
パレット。 t で虹色、uTint で色味を回す（入門 D）。
vec2 uv = (gl_FragCoord.xy*2.0 - uResolution) / uResolution.y;
float t = uTime * uSpeed;
座標と時間。 中心0・アスペクト保持の uv。t は速度付きの時間。
vec2 q = vec2( fbm(uv + t),                fbm(uv + vec2(5.2,1.3) - t) );
vec2 r = vec2( fbm(uv + uWarp*q + uTilt*uTiltK + vec2(1.7,9.2) + 0.15*t),
               fbm(uv + uWarp*q + uTilt*uTiltK + vec2(8.3,2.8) - 0.12*t) );
float f = fbm(uv + uWarp*1.25*r + uTilt*(uTiltK*0.5));
domain warping＝うねりの核心。 まず q で uv を歪め、その結果 r でさらに歪め、最後に f を取る。「ノイズの座標をノイズでずらす」入れ子が、まっすぐでない有機的な流れを生む。uTilt を混ぜることで傾きが流れの向きに効く。
vec3 inkCol = pal(f*0.6 + length(r)*0.35 + t*0.1);
vec3 col = mix(uBase, inkCol, smoothstep(uInk, 0.9, f));
色を置く。 f の値に応じて地色とインク色を混合。smoothstep(uInk,0.9,f) がインクの滲み境界（uInk で広がりを調整）。
float edge = pow(1.0 - abs(f - 0.55)*2.0, 3.0);
col += uGlow * edge * inkCol;
float vig = 1.0 - 0.35*dot(uv*0.7, uv*0.7);
col *= clamp(vig, 0.0, 1.0);
gl_FragColor = vec4(col, 1.0);
仕上げ。 f≈0.55 の等高線だけを光らせて膜の縁の発光に。最後に vig（周辺減光）を掛けて出力。
demo 2

万華鏡 — 極座標と対称のトリック

kaleidoscope.html。座標を「角度と半径」で考えると、対称や回転が一気に書ける。

mat2 rot(float a){ float c=cos(a), s=sin(a); return mat2(c,-s,s,c); }
vec2 uv = (gl_FragCoord.xy*2.0 - uResolution) / uResolution.y;
uv = rot(uTime*0.1*uSpeed + uTilt.x*1.5*uTiltK) * uv;
全体を回す。 rot は回転行列。時間と傾きで uv 全体を回転させる。
float a = atan(uv.y, uv.x);   // 角度（-π..π）
float r = length(uv);         // 半径（中心からの距離）
float span = 6.28318 / uSeg;  // 1セクタ分の角度
極座標へ。 atan で角度、length で半径。uSeg（対称数）で円を何分割するか決める。
a = mod(a, span);        // 角度を1セクタ内に畳む
a = abs(a - span*0.5);   // セクタ中心でミラー → 対称
これが万華鏡。 mod で角度をセクタ内に折り返し（＝同じ模様を放射状に複製）、abs でセクタ内も鏡像化。この2行だけで対称が生まれる。
vec2 p = vec2(cos(a), sin(a)) * r;   // 畳んだ角度で座標に戻す
p *= uScale;
p += vec2(uTime*0.15*uSpeed, 0.0) + uTilt*0.6*uTiltK;
模様用の座標。 折り返した角度＋半径で xy に戻し、スケールと移動（時間・傾き）を加える。
vec2 g = fract(p) - 0.5;   // タイル化：各セルの中心を0に
float d = length(g);
float ring = sin(d*uRingFreq - uTime*2.0*uSpeed);
float edge = smoothstep(0.0, 0.12, abs(ring));
セル内の模様。 fract で空間を格子に区切り、各セル中心からの距離 d に sin をかけて同心リングを作る。uRingFreq でリングの細かさ。
vec3 col = 0.5 + 0.5*cos(6.28318*(r*0.5 + uTime*0.05*uSpeed + uTint));
col *= 0.4 + 0.6*edge;
col *= 1.0 - 0.25*r;
gl_FragColor = vec4(col, 1.0);
色。 半径で虹色（uTint で色味）。リングの線 edge で明暗を付け、外周を少し暗く。
demo 3

反応拡散 — 状態をテクスチャに持つシミュレーション

reaction-diffusion.html。これまでと違い3つのパスを使う：種まき(SEED)→更新(SIM)→表示(DISP)。状態（2種の薬品 A・B の濃度）はテクスチャに保存し、毎フレーム更新する。

// SEED：初期状態。A=1で全面、Bをまばらに置く
float B = 0.0;
if(hash(floor(vUv*10.0)) > 0.93) B = 1.0;
if(distance(vUv, vec2(0.5)) < 0.04) B = 1.0;
gl_FragColor = vec4(1.0, B, 0.0, 1.0);   // R=A, G=B
種まきパス。 テクスチャの R チャンネルに A、G に B を入れる。最初は A=1（全面）に、B をまばらな点と中央に置く。
// SIM：隣接セルを読む
vec2 S(vec2 o){ return texture2D(uState, vUv + o*uTexel).xy; }
vec2 lap = 0.2*(S(vec2(1,0))+S(vec2(-1,0))+S(vec2(0,1))+S(vec2(0,-1)))
         + 0.05*(S(vec2(1,1))+S(vec2(-1,1))+S(vec2(1,-1))+S(vec2(-1,-1)))
         - 1.0*c;
ラプラシアン（拡散）。 uTexel＝1ピクセル分の幅。上下左右＋斜めの隣を読み、重み付き平均と中心の差をとる＝「まわりへ滲む量」。この係数（0.2 / 0.05 / -1）が標準値で、強すぎると“凍りつく”不具合になった箇所。
float A = c.x, B = c.y;  float Da=1.0, Db=0.5;
float reaction = A*B*B;
float A2 = A + (Da*lap.x - reaction + uFeed*(1.0 - A));
float B2 = B + (Db*lap.y + reaction - (uKill + uFeed)*B);
反応（Gray-Scott）。 「A + 2B → 3B」を A*B*B で表す。B は A を食べて増え（reaction）、一定割合で消え（kill）、A は供給される（feed）。uFeed/uKill の組でサンゴ・迷路・分裂など別模様になる。
vec2 bp = vec2(0.5) + uTilt*0.4;
if(distance(vUv, bp) < uBrush) B2 = 1.0;                 // 傾きで動く筆
if(h21(floor(vUv*200.0) + floor(uSeed*3.0)) > 1.0 - uSeedRate) B2 = 1.0;  // 種まき（枯渇防止）
gl_FragColor = vec4(clamp(A2,0.0,1.0), clamp(B2,0.0,1.0), 0.0, 1.0);
入力と書き戻し。 傾きの位置に B を足す（＝触ると模様が生える）。少量の自動種まきで枯渇を防止。clamp して次の状態を書き出す。
// DISP：表示パス。B濃度を色に
float Bv = texture2D(uState, vUv).y;
float v = smoothstep(0.0, 0.35, Bv);
vec3 col = mix(uColLo, uColHi, v) + smoothstep(0.25,0.5,Bv)*vec3(0.45,0.1,0.6);
表示。 シミュレーション結果（B の濃度）を地色→模様色に変換するだけ。計算と見た目を分離している。
// JS：ping-pong（2枚のテクスチャを交互に）
for(let k=0;k<s.steps;k++){
  s.sim.uniforms.uState.value = s.rts[s.read].texture;          // 今の状態を読み
  renderer.render({ scene: s.simMesh, target: s.rts[s.read ^ 1] }); // 次の状態を別テクスチャに書く
  s.read ^= 1;                                                   // 読み書きを入れ替え
}
記憶の正体。 同じテクスチャを読みながら書くことはできないので、2枚を用意して交互に使う（ping-pong）。これを1フレームで steps 回繰り返して時間を進める。「前の状態から次を作る」＝シミュレーションはこの構造で成り立つ。
demo 4+

残りのデモ — 勘所だけ

基本パターンは上の3本の組み合わせ。各デモで「新しく効いている1行」を抜き出す。

トンネル — 半径の逆数で奥行き

1/r が「遠近」を作るキモ。中心(r小)ほど depth が大きく＝遠くまで詰まって見える。

コースティクス — 反復で光の網

座標を sin/cos で繰り返し折り曲げて干渉模様を作り、pow で明暗を締める。uContrast が光の鋭さ。

水中の生き物 — 関数化して“ずらして再サンプル”＝屈折

勘所は2つ。①既存シェーダーを関数化すれば、座標をずらして呼ぶだけで「レンズ越し」が作れる（屈折・熱揺らぎ・水滴などに応用可）。②位置と速度を JS から渡す＝Breakoutと同じハイブリッドのインタラクション。慣性つきの追従で“泳ぎ”の質感を出している。

グリッチ — 行ごとにズラす＋RGB別取り

カメラ版は scene() が texture2D(uTex, ...)＝実映像。同じグリッチ式を、手続き模様にもカメラにもかけられるのがポイント。

粒子 — 頂点シェーダーで位置計算＋流れ場

分岐ツリー — JSで形を生成、GPUで太く描く

形の生成はJS（再帰）、描画はGPU。役割分担の好例。

Breakout — SDF描画＋JSロジック＋データテクスチャ

描画はシェーダー、物理・スコアは JS、ブロックの状態はデータテクスチャでGPUへ。学習ノートの「ハイブリッド」そのもの。
