文字が育つ反応拡散

smapho_demo / 案 #3 — 既存の reaction-diffusion デモを文学テクストで種付けする

ライブシミュレーション · Gray-Scott t = 00:00
速度 4.0×

「羅生門」全文を 4 本のトラック（右・右中・左中・左、各 40px 幅）に並べ、各トラックが独立した 静止 → 立ち上がり → 拡散 → 再集結 サイクルを 1/4 サイクルずつズラして 回す。 再集結は セルごとに別タイミングで target に snap させ、文字セルから先に粒として立ち上がる — クロスフェードではなく「粒が集まって字になる」見え方を狙う。 速度スライダーで読み進みのテンポを 0.5×〜4× で調整（既定 2×）、終端で先頭へループ。

進行

各トラックは独立な 28 秒サイクル（HOLD 4 / RAMP 10 / DISSOLVE 12 / REFORM 2、stagger = 7s。速度スライダーで wall-clock 上の長さを 0.5×〜16× 調整）。 HOLD+RAMP = 14 = 2×stagger なので、常に 2 トラックが hold か ramp（=可読）、残り 2 トラックが拡散か再集結のオーバーラップ構成。 sim は常時全速力で回し、位相は hold=lock 1.0（背景セルは ×BG_LOCK_MUL=0）/ ramp=lock 1−p³ / dissolve=lock 0 / reform=per-cell snap で表現する。

  1. 4 s

    静止 — 行を読む（lock = 1）

    毎フレーム target の値で完全に上書き。sim はトラックの外側で進むが、このトラックは text のまま留まる。

  2. 10 s

    立ち上がり（lock = 1−p³）

    lock が cubic ease-in で 1 → 0 に解放される。最初は固く保たれ、終盤で急に sim に明け渡される。画数の細い部分から先に変質する。

  3. 12 s

    拡散（lock = 0）

    このトラックは sim 任せ。プリセットの f, k に従って固有の崩れ方をする。mitosis なら胞子状、coral なら細胞状、spots なら孤立点、chaos なら不規則な滲み。

  4. 2 s

    再集結 — per-cell snap で「粒が集まる」

    そのトラックの x-range に 次の行のグリフを target として焼き込み、各セルに 0..REFORM_S の snap 閾値を乱数配布する。文字セル（targetV 明）は前半 55% に、背景セルは後半 60% にバラまく。phaseElapsed が閾値を超えたセルは target に snap し、未 snap セルは sim 任せで churning。結果、セルが個別タイミングでカチカチ立ち上がり、文字が先に組み上がってから背景が落ち着く。一斉クロスフェードを避けるための要。

  5. ↻

    4 トラック並列 / 1/4 サイクルズレ / global counter

    初期 hold を 0 / CYCLE/4 / 2·CYCLE/4 / 3·CYCLE/4 で延長して定常状態へ。reform を抜ける順に nextGlobalLine から行番号を払い出すので、読み順 0→1→2→3→… が保持される。最終行の reform 完了で先頭へループ。本文は青空文庫の旧字旧仮名版から起動時に fetch、約 410 行に分割。

素材

或日の暮方の事である。一人の下人が、羅生門の下で雨やみを待つてゐた。 芥川龍之介「羅生門」 (1915) · 青空文庫 No.879 · 旧字旧仮名

青空文庫の HTML は Shift_JIS、CORS も不可。curl | iconv -f SHIFT_JIS -t UTF-8 で取り、Python で <ruby> と ［＃…］ 等を剥がし concepts/texts/rashomon.txt として repo に同梱（パブリックドメイン）。起動時に fetch、句点で文単位 → 長文は読点で更に分割し、目標 18 字以下の読み取り単位に整形（実測 約 410 行・平均 14 字）。

実装

既存の demos/reaction-diffusion.html をベースに、以下を足すだけで成立する。Ping-Pong レンダリングと自動種まきの足回りは既に書いてある。

  • text-to-texture パス: 当初 SVG<foreignObject> + writing-mode: vertical-rl でブラウザ組版に丸投げを試みたが、foreignObject 入り画像は canvas を cross-origin tainted 化し getImageData が SecurityError になる仕様だったため断念。代わりに Canvas2D fillText で 1 字ずつ縦に流しつつ、縦書きで崩れる文字種だけを補正: 「」『』 は U+FE41〜FE44 の縦書き専用形に差替え、、。，． は字面の右上へ平行移動 (+0.45fs, -0.5fs)、― — ー 〜 〜 ‐ は 90° 時計回り回転。羅生門本文に出現するこれら 4 種で網羅される。
  • per-cell snap による再集結: dissolve→reform 移行時に各セルへ snapMoment[i] ∈ [0, REFORM_S] を乱数配布。targetV[i] > 0.4 の文字セルは前半、背景セルは後半に snap させ、粒が個別タイミングで固まり、文字が先に立ち上がる挙動を作る。一斉 lerp の crossfade 感を消す要。
  • 速度スライダー: フレーム毎の dt に倍率（0.5×〜16×、既定 4×）を掛けて sim 時間に変換。サイクル長もシミュレーションも同じ係数でスケールするので、テンポを変えても各位相の比は保たれる。
  • 禁則処理: 行分割後に 、。」』） ぁぃぅ ャュョ などが行頭に来た場合は前行末に押し戻し、「『（ などが行末に来た場合は次行頭へ移動。「閉じカギ括弧だけ行頭」「、だけの行」等の不自然な分割を解消。
  • crisp ハイライト: hold/ramp の lock 強度（cubic ease-in で 1→0）を文字セル（targetV > 0.4）の RGB に乗算（最大 1.6×）。読んでいる行の文字が周囲よりはっきり光って見える。
  • 挙動プリセット: mitosis / coral / spots / solitons の f,k ペア切替。文字の崩れ方が劇的に変わる（上のキャンバスでも試せます）。
  • paramPanel 追加項目: text:作品ID, range:フォントサイズ, button:再シード, button:次の節へ。値は URL ハッシュへ自動保存される（既存仕様）。
  • カラーマップ: 既存 reaction-diffusion デモは emerald → purple。本案は素材に合わせて苔の闇から螢光の青磁へ。朽葉色 #b5573a を縁取りアクセントに。
このページの canvas について ／ 実際に Gray-Scott を走らせている（160×360 グリッド、Karl Sims 9点ラプラシアン、Canvas2D + Float32Array、180 sps × 速度倍率（位相は full、sim は 8× で頭打ち））。 本番のデモは WebGL2 の半精度 RT で同等以上の解像度を出します。 日本語縦書きは Canvas2D fillText で 1 字ずつ流し、縦書きで崩れる文字（句読点・カギ括弧・長音/ダッシュ）だけを補正テーブルで明示的に処理。foreignObject 方式は canvas を tainted 化して getImageData が拒否されるため不可。
