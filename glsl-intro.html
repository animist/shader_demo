GLSLの読み方入門

シェーダーのコードは、慣れると「短い数式の集まり」として読める。ここでは、デモに出てくる書き方だけに絞って“読める”ようにする。書けることより、まず読めること。

A

まず雰囲気：C風で、型に厳しい

GLSL は C によく似た小さな言語。大きな特徴が2つ。(1) 型に厳しい（1 は整数、1.0 は小数。混ぜると怒られる）。(2) ベクトルが基本型（座標も色も vec）。

swizzle（スウィズル）：成分は .x .y .z .w でも .r .g .b .a でも取れ、並べ替えもできる。

ベクトルの計算は成分ごと（component-wise）。vec3*vec3 は各成分の掛け算、a+b も各成分の足し算。だから色や座標を一気に処理できる。
B

座標の基本：ピクセル位置から uv を作る

フラグメントシェーダーには gl_FragCoord.xy（いま計算中のピクセルのピクセル座標）が与えられる。これを使いやすい範囲に正規化するのが第一歩。デモでは主に2通り。

(2) は インクや万華鏡で使う「中心0」座標。円や対称形を歪ませず描ける。(1) はグリッチやBreakoutのように画面全体を 0..1 で扱いたいときに使う。

C

よく出る組み込み関数

シェーダーは関数のボキャブラリが命。デモに出てくるものだけ表にする。これだけ知れば大半が読める。

関数意味デモでの使い所
mix(a,b,t)a と b を t(0→1)で線形補間色の混合：根元色→先端色（ツリー）
clamp(x,lo,hi)範囲に収める色や値の暴走防止
step(e,x)x≥e で 1、未満で 0（段差）「この行だけ割れる」判定（グリッチ）
smoothstep(e0,e1,x)e0→e1 でなめらかに 0→1境界のアンチエイリアス・滲み（要 e0<e1）
fract(x)小数部（0..1 の繰り返し）タイル化・ノイズ・万華鏡のセル
floor(x) / mod(x,y)切り捨て / 剰余セル番号・角度の折り返し
abs(x)絶対値ミラー（左右対称）
length(v) / distance(a,b)長さ / 2点間距離円・ボールのSDF、傾き量
normalize(v) / dot(a,b)単位ベクトル化 / 内積方向・線の太さの法線・ハッシュ
pow(x,y)累乗コントラスト（コースティクス）・先細り
sin/cos / atan(y,x)三角関数 / 角度を求める波・揺れ・パレット・極座標
texture2D(s,uv)テクスチャから色を取る反応拡散・カメラ・Breakoutのブロック状態

smoothstep は特に頻出

「ある値をなめらかな 0→1 のフェードに変える」関数。境界をギザギザにせず描くのに使う。step は段差、smoothstep はなめらか、と覚える。

D

色を作る定番：cos パレット

デモのいたる所に出てくるこの1行。これは cos で3チャンネルの位相をずらして虹色を作るイディオム（IQ のパレット）。

  • cos は −1〜1 → 0.5+0.5*cos で 0〜1（色の範囲）に収める
  • 6.28318 = 2π。t が 1 増えると一周（色が一巡）
  • vec3(0, .33, .66) で R・G・B の位相をずらす → 同じ t でも3色が違う位置 → 虹
  • t に半径・時間・ノイズ値を入れると、それに応じて色が流れる

デモの「パレット」「色」スライダーは、この vec3(...) の位相をいじっている。

E

制御フロー・入力・精度

if と for は使えるが…

分岐もループも書けるが、ループの回数は基本「定数」にする（モバイルGPUの制約）。discard; はそのピクセルを描かずに捨てる（粒子を丸く切り抜くのに使用）。

入力 3種

  • uniform … 全ピクセル共通。uTime（時間）, uResolution（解像度）, uTilt（傾き）, ⚙のパラメータ。JSから毎フレーム送る。
  • attribute → varying … 頂点ごとの値が、ピクセルへ補間されて届く（例：粒子の vLife）。
  • sampler2D + texture2D … 画像やデータをテクスチャとして読む。

precision（精度）宣言

フラグメントシェーダーは先頭で精度を宣言する。highp（高精度）が使えない古い端末向けに、デモは保険を入れている（→ 学習ノートの「落とし穴」参照）。

F

“読む”ときの手順

  1. 入力（uniform）を確認：何が外から入ってくるか
  2. 座標 uv の作り方を見る：中心0か、0..1か
  3. main の最後 gl_FragColor から逆算：最終的に何色を出すか → その色がどう作られたか上にたどる
  4. 関数（noise/fbm/box/pal 等）は“部品”として名前で理解：中身は後で
